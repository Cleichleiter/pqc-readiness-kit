Sample Outputs



This directory contains sanitized example outputs generated by the toolkit for demonstration and documentation purposes.



Sample files are provided to:



Illustrate output structure and fields



Support documentation screenshots and walkthroughs



Enable offline review without running collectors



Avoid committing environment-specific or sensitive runtime data



What belongs here



Curated example outputs with .sample.\* filenames



Data that has been reviewed to ensure it contains no sensitive information



Files intended for reference only



Examples:



crypto\_inventory.sample.json



findings.sample.csv



What does not belong here



Live runtime outputs



Organization-specific or production data



Automatically generated files that have not been reviewed



Naming Convention



Sample files should use the format:



<artifact>.<purpose>.sample.<ext>



This makes it explicit that the file is not a live output.
